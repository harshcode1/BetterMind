Issue: Doctor Verification Status Not Updating

There's an issue where doctors are still seeing the verification pending page even though their verification status is set to true in the database. This happens because:

  1. The verification status might be updated in the doctors collection but not in the users collection
  2. The user's session might be using cached data that doesn't reflect the updated verification status
Note: The tools on this page will help you fix these issues by ensuring the verification status is correctly updated in both collections and by refreshing the user's session.

How to Fix the Issue

1 Force Verify the Doctor

Use the Force Verify tool to update the verification status for a specific doctor in both the doctors and users collections.

Go to Force Verify Tool
2 Log Out and Log Back In

After updating the verification status, the doctor needs to log out and log back in to refresh their session with the updated verification status.

Go to Logout Page
3 Verify the Fix

After logging back in, the doctor should be redirected to the doctor dashboard instead of the verification pending page.

Technical Details

The Force Verify tool works by:

  1. Finding the doctor record in the doctors collection
  2. Updating the verification status to true in the doctors collection
  3. Using the userId from the doctor record to find the corresponding user record
  4. Updating the verification status to true in the users collection

This ensures that the verification status is correctly updated in both collections, which is necessary for the doctor to be recognized as verified throughout the application.
